# Relationship

## 관계 정의: 모델
- 회원이 있고 회원의 게시글이 있는기본적이 관계
```python
# model side
## user side
items = relationship("DbArticle", back_populates="user") #유저 측에서, 자신이 쓴 글들의 목록을 일대다 관계로 매핑

## article side
user_id = Column(Integer,ForeignKey("users.id"))
user = relationship("DbUser", back_populates="items") # 글 측에서 작성자를 다대일 관계로 매핑.
```
## 관계 정의: 스키마
```python
#유저 내부의 아티클만을 위한 디스플레이 모델을 따로 만들어주는것이 일반적
class Article(BaseModel):
    title: str
    content: str
    published: bool

    class Config:
        orm_mode = True

# 실제 유저 디스플레이
class UserDisplay(BaseModel):
    items: List[Article] = []
    ...

# 아티클 내부의 유저만을 위한 디스플레이 모델을 따로 만들어주는 것이 일반적.
class User(BaseModel):
    id: int
    username: str

    class Config:
        orm_mode = True
# 실제 아티클 디스플레이
class ArticleDisplay(BaseModel):
    user: User
    ...
```
